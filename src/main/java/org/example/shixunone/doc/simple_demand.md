# 分销商城系统需求分析报告
---

## 1. 项目引言

### 1.1 项目背景
本项目旨在利用Spring Boot、Vue3、Redis及RabbitMQ等主流技术栈，构建一个分销商城系统。“商品展示-模拟下单-异步返利”的核心闭环，验证全栈技术在实际业务中的应用。

### 1.2 建设目标
1.  **技术验证**：实现前后端分离架构，验证Redis会话管理与RabbitMQ异步解耦在实际业务中的可行性。
2.  **核心逻辑**：实现“模拟购买 -> 确认收货 -> 异步计算佣金”的业务流。
3.  **教学演示**：构建一个可运行、可交互的系统原型，用于展示全栈开发流程。

---

## 2. 用户角色定义

系统主要包含两种核心角色。

| 角色 | 别名 | 核心职责 | 权限说明 |
| :--- | :--- | :--- | :--- |
| **管理员** | 后台运营 | 数据维护 | 商品发布、查看订单列表、配置佣金比例。 |
| **普通用户** | 消费者/推广员 | 购物/获利 | 浏览商品、模拟下单、确认收货、查看余额、生成邀请链接。 |

## 3. 功能需求详解

### 3.1 用户中心模块
*   **注册/登录**：
    *   支持手机号注册/登录。
    *   **Redis应用**：登录成功后生成Token，利用Redis存储Token与用户ID的映射（设置过期时间），实现无状态会话管理。
*   **个人资产**：
    *   **模拟钱包**：仅展示余额数值，不涉及真实银行卡绑定。
    *   **模拟提现**：点击“申请提现”仅扣除余额并提示成功，不涉及财务审核与打款。

### 3.2 商品中心模块 (管理员端)
*   **商品管理**：
    *   **发布商品**：录入商品名称、价格、库存。
    *   **设置佣金**：配置该商品的固定返利金额（如：购买此商品返利 2 元）。
*   **商品列表**：展示所有商品，供用户浏览。

### 3.3 订单与交易模块
*   **模拟下单**：
    *   **立即购买**：跳过购物车，直接生成订单。
    *   **状态流转**：`待支付` -> `待收货` -> `已完成`。
    *   **支付模拟**：点击“支付”按钮直接跳过支付环节，状态变更为“待收货”。
*   **确认收货**：
    *   用户点击“确认收货”。
    *   **RabbitMQ应用**：后端不直接计算佣金，而是向消息队列发送一条“OrderPaidEvent”消息，包含订单ID。

### 3.4 分销与返利模块 (核心)
*   **关系绑定**：
    *   **邀请链接**：用户生成链接 `?invite_id=1001`。
    *   **绑定逻辑**：新用户首次访问时，系统记录其“上级ID”（存入数据库字段 `parent_id`）。
*   **异步返利 (RabbitMQ)**：
    *   **生产者**：订单确认收货时，发送消息到队列。
    *   **消费者**：后台监听队列，收到消息后执行：
        1.  查询订单详情。
        2.  查询上级推广员 ID。
        3.  **MySQL事务**：更新上级用户的余额（`balance = balance + commission`）。
*   **收益记录**：展示用户获得的佣金明细。

### 3.5 后台管理模块 (管理员)
*   **仪表盘**：展示今日订单数、用户总数。
*   **商品管理**：商品的增删改查，调整佣金比例。
*   **订单管理**：查看所有模拟订单的状态。

## 4. 核心业务流程图 (逻辑描述)

### 4.1 用户登录与鉴权流程
1.  用户输入账号密码。
2.  后端验证通过，生成 UUID 作为 Token。
3.  **Redis**：`SET Token 用户ID EX 3600` (存入Redis，有效期1小时)。
4.  前端存储 Token，后续请求携带 Token 在 Header 中。
5.  后端拦截器校验 Redis 中是否存在该 Token，存在则放行。

### 4.2 异步返利结算流程
1.  用户点击“确认收货”。
2.  **Spring Boot**：更新订单状态为“已完成”。
3.  **RabbitMQ**：`rabbitTemplate.convertAndSend("commission.queue", orderId)`。
4.  **消费者监听**：收到 orderId，开启数据库事务。
5.  **计算**：查询订单金额 -> 查询上级ID -> 计算佣金。
6.  **MySQL**：`UPDATE user SET balance = balance + 2 WHERE id = parentId`。
7.  提交事务。

---

## 5. 数据需求 (核心实体)

1.  **Users (用户表)**: ID, username, password, balance (余额), parent_id (上级ID), create_time。
2.  **Products (商品表)**: ID, name, price, stock, commission_amount (固定佣金金额)。
3.  **Orders (订单表)**: ID, user_id, product_id, amount (金额), status (0待收货/1已完成), create_time。
4.  **Rebate_Records (返利记录表)**: ID, user_id (推广员), order_id, money, create_time。

---

## 6. 非功能需求

1.  **技术架构**：
    *   **前后端分离**：前端 Vue3 跨域访问后端 Spring Boot。
    *   **中间件应用**：强制使用 Redis 存储会话，强制使用 RabbitMQ 处理返利逻辑。
2.  **数据一致性**：
    *   仅需保证 MySQL 基础事务（ACID），确保余额更新不会出错。
3.  **安全性**：
    *   **模拟安全**：不涉及真实资金，无需复杂的风控系统。
    *   **Token校验**：确保未登录用户无法访问核心接口。

## 7. 开发阶段建议

1.  **第一阶段：环境搭建**
    *   安装 MySQL, Redis, RabbitMQ。
    *   初始化 Spring Boot 项目（Web, Data JPA/MyBatis, Redis, AMQP）。
    *   初始化 Vue3 项目。
2.  **第二阶段：基础CRUD**
    *   实现商品列表查询。
    *   实现用户注册登录（集成 Redis）。
3.  **第三阶段：核心逻辑**
    *   实现模拟下单。
    *   配置 RabbitMQ，实现发送与接收消息。
4.  **第四阶段：联调**
    *   前后端联调，打通全流程。
